String - обьект, последовательность символов

Способы создания строк:
1. Строковый литерал
Проверяет - есть ли строка в пуле, возрвращает - если есть, добавляет - если нет
String s1 = "Hello";
2. Оператор new
Даже если есть обьект в пуле, создается новый обьект в куче
(Вообще литерал добавляется в пул - а потом копируется и создается обьект в куче)
String s2 = new String("Hello");
3. Конструкторы
Создает строки из массива символов или байт с помощью встроенных в класс String конструкторов(только через new)
char[] chars = {'H', 'e', 'l', 'l', 'o'};
String s3 = new String(chars);
еще можно байт-массив

Строки в java неизменяемы
Поэтому любая "изменяющая" операция - создает новый обьект String
Нужно для:
- Безопасности
- Кэширования и производительности
- Многопоточности

Литеральные и интернированные строки (и константы) хранятся в пуле, который в свою очередь находится в куче
Строки созданные new или динамически (считывание файла) - обычне обьекты в куче
intern() - проверяют есть ли строка в пуле, возрвращает - если есть, добавляет - если нет
Интернирование нужно как раз для обьектов в куче

Сравнение строк:
1. equals() - сравнивает содержимое строк
2. == - сравнивают ссылки на обьекты

Конкатенация строк:
1. plus
фактически вызывает СтрингБилдер под капотом
поэтому в цикле лучше явно использовать СтрингБилдер, чтобы не создавать новый обьект в каждой итераци, инче в цикле вылезет квадратичная сложность
Опять же - если выполняется на этапе компиляции (литералы и константы), то попадает в пул, иначе создается новый обьект в куче
2. concat()
Создает новый обьект в куче
не нуль безопасен
?немного быстрее "+"?
3. String.join()
Нуль безопасен, лаконичен, работает с любыми итерируемыми обьектами
Но всегда создает новую строку, даже если передан один элемент
4. StringJoiner()
Есть префикс и суффикс, использует Стринг Билдер
Полезен для JSON-массивов и Stream API потоков

- Просто склеить 2 строки → +
- Собрать строку в цикле → StringBuilder или StringJoiner
- Объединить коллекцию/массив с разделителем → String.join() или стримы
- Нужны префикс/суффикс → StringJoiner или Collectors.joining(..., ..., ...)

1. Длина строки
length()
2. Доступ к символам
charAt(index)
3. Поиск подстроки\символа
indexOf()
lastIndexOf()
contains()
startsWith()
endsWith()
4. Извлечение подстроки
substring(int beginIndex, int endIndex)
5. Изменение регистра
toLowerCase()
toUpperCase()
6. Удаление пробелов
trim()
strip()
stripLeading()
stripTrailing()
7. Замена
replace()
replaceAll()
replaceFirst()
8. Разделение строк
split()
