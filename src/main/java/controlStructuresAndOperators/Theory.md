Управляющие конструкции - основа структурного программирования

Три базовые структуры:
- Следование
- Ветвление
- Повторение

По теореме Бёма - Якопини, любая программа может быть написанна с использованием только этих трех конструкций

УСЛОВНЫЙ ОПЕРАТОР if - else
Опеределяет true или false в if, если true - выполняет вложение, если false - переходит ниже

Тернарный оператор
Условие ? Блок для true : Блок для false
Оба выражния должны быть одного совместного типа

Тип условия только boolean

Область видимости внутри < снаружи

if ("test".equals(str)) { ... } - нуль проверка, сработает даже если str == null, можно писать двойное условие с отдельной нуль проверкой

Дешевые и вероятные проверки раньше, дорогие и редкие позже

else относится к ближайшему if, поэтому чтобы избежать проблем, просто достаточно все явно прописывать скобками

Частный случай паттерна "Ранний возврат" - "Стражиники" - сначала проверки, потом основная логика

Каскадный поиск лучше выполнять через switch

ОПЕРАТОР switch

НЕ допускаются для использования в кейсах:
long, float, double, boolean

Есть три стратегии компиляции:
- LookupSwitch - для разреженных значений 
- TableSwitch - для плотных диапазонов 
- StringSwitch - через хэш-код 

Чтобы избежать проваливания, стоит каждый кейс break

Зачастую работает быстрее чем if-else

Стоит использовать новый стрелочный синтаксис

ЦИКЛЫ

- for - известно кол-во итераций
- while - выполняется пока соблюдается условие
- do-while - пост-условие, когда нужна минимум одна итерация цикла
- for-each - синт сахар для переобора итерируемого обьекта

Операторы управления потоком:
- return - возвращает и прерывает
- break - прерывает
- continue - переходит к следующей итерации 
Так же, для операторов потока можно использовать метки, например что бы break прерывал сразу внешний цикл
...: - начальная метка
break ...; - остановка по метке
