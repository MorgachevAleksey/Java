Многопоточность - одновременное выполнение нескольких задач

Поток - наименьшая единица выполнения, независимо планируемая системой

Многопоточность нужна для:
- Параллелизма - выполнять несколько задач одновременно
- Асинхронности - не блокировать основной поток при долгих операциях
- Утилизации ресурсов - один поток ждет ввод\вывод, второй вычисляет

Два способа реализации:
1. Наследование класса Thread - используется для тонкого управления конкретным потоком
Но т.к. нельзя наследоваться от нескольких классов, может быть менее преподчтительным способом (возможно вовсе устаревшим)
2. Реализация интерфейса Runnable
Рекомендуемый способ т.к. отделяет задачу (Runnable) от механизма ее выполнения (Thread)
3. Так же существуют лямбда-выражения

Состояния потока (жизненный цикл):
- NEW - поток создан, но еще не запущен
- RUNNABLE - поток готов к выполнению и может выполняться (после start()). Включает в себя как выполнение на процессоре, так и ожидание очереди планировщика
- BLOCKED – поток заблокирован и ждёт монитор (вход в synchronized блок/метод)
- WAITING – поток ждёт бесконечно, пока другой поток не вызовет notify()/notifyAll() (методы wait(), join() без таймаута)
- TIMED_WAITING – поток ждёт в течение заданного времени (sleep(ms), wait(timeout), join(timeout))
- TERMINATED – поток завершил выполнение (run() закончился или было исключение)

Переходы:
- new -> start() -> runnable 
- runnable -> terminated (когда run завершился)
- runnable -> blocked/waiting/timed_waiting при вызове соответствующих методов 
- По окончании ожидания или получения монитора – возврат в runnable.


implements - ключевое слово для исполнения контракта (реализации всех методов) интерфейса
Runnable - интерфейс для задачи выполняемой потоком или пулом потоков
