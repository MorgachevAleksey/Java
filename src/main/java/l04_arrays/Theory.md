Массив - контейнер, хранящий некоторое фиксированное кол-во элементов одного типа
В java - это не отдельный класс, а втроенная в язык структура
Элементы доступны по индексам, начиная с 0
Доступ к элементу массива — O(1)
Массивы примитивов хранятся в памяти локальными блоками с элементами подряд, поэтому доступ очень быстрый благодаря кэшу процессора
Поиск - O(n)
Копирование - O(n)
Сортировка - O(n log n).

- Фиксированный размер
- Однотипность
- Индексация с 0
- Обьект - хранится в куче

Обьявление массива
Предпочтительный способ обьявления:
int[] numbers
String[] names

но можно и так:
int numbers[]

На этапе обьявления память под массив не выделяется, а значение - по-умолчанию, null

Создание массива - выделение под него памяти
Массив создаются с помощью оператора new, с указанием типа и размера
numbers = new int[10]
В этот момент в куче создается обьект-массив, а все элементы заполняются значениями по умолчанию

Можно обьеденить процесс обьявления и создания
int[] numbers = new int[10]

Инициализация массива:
1ый способ - литерал массива, фактически это синтаксический сахар для второго способа
String[] names = {Aleksey, Alexander}

2й способ - анонимный массив
String[] names = new String[]{Aleksey, Alexander}
вариант использования:
printArray(new int[]{1, 2, 3})

3ий способ - поэлементное присванивание 
int[] numbers = new int[5]    

У каждого массива есть публичное финальное поле length - размер
int size = numbers.length;
Поле изменить нельзя. Так как это поле, в отличие от строкового метода - скобки не ставятся

Типы массивов:
1. Массив примитивов - хранит непосредственные значения в непрерывной области памяти
Каждый элемент занимает столько памяти, сколько требует тип
2. Массив ссылочных типов - хранит ссылки на обьекты
Размер ссылки зависит от JVM - 4 или 8 байт
Сами обьекты лежат в отдельной куче

Т.к. массив - это обьект, то при передаче, передается копия ссылки, а не сам массив
Ссылочная переменная массива, может быть присвоена другому массиву или null

В стандартной библиотеке есть класс-утилита Arrays
В нем находятся перегруженные статические методы для работы с массивами, имеющими фиксированный размер

Основные методы:
1. Arrays.fill - заполняет массив или его часть указнным значением
Изменяет исходный массив
2. Копирование массива
2.1. System.arraycopy - копирование данного массива в целевой
Аргументы: данный массив, с какого элемента, целевой, с какого, сколько элементов
Самый быстрый
2.2. Этот же метод но в обертке Arrays.copyOf -  сразу создает копию данного
Аргументы: данный массив, сколько
Самый удобный и читаемый
2.3. Есть еще Arrays.copyOfRange - до какого то элемента
Когда нужна часть
2.4. Системный clone()
   int[] original = {1, 2, 3};
   int[] copy = original.clone();
3. Arrays.equals \ Arrays.deepEquals - поэлементное сравнение для одномерных\мнногомерных массивов 
4. Arrays.sort - сортировка элементов, имзеняет исходный массив
Есть компаратор для обьектов (не для примитивов) для изменения порядка сортировки 
5. Arrays.binarySearch() - для быстрого поиска элемента в отсортированном массиве 
6. Arrays.toString \ Arrays.deepToString - возвращает строковое представлние
7. Arrays.setAll
8. Arrays.parallelPrefix

Выход за пределы массива вызывает исключение ArrayIndexOutOfBoundsException (подкласс RuntimeException)

Можно перебирать:
1. Счетчиком
2. for-each - способ короче, но возвращает копию элемента а не элемент, поэтому не подойдет для изменения элементов !примтивов!

Многомерные массивы

В java многомерные массивы - это массивы массивов

Внутренние массивы создаются сразу
int[][] matrix = new int[3][4];

Зубчатый массив - массив в котором внутренние массивы имеют разную длину
Его можно инициилизоровать сразу
int[][] jug = 
{
    {1, 2}, 
    {1, 2, 3}
}
Или сначала внешний, затем внутренние
int[][]jug = new int[2][];
jug[0] = new int[2];
jug[1] = new int[3];

Доступ к элементам зубчатого массива
jug[0][1] = 30;
int x = jug[0][1];

jug.length - 2
jug[1].length - 3

Обход - вложенными for или for-each

Копирование:
1. Создать массив с таким же количеством внешних массивов
2. Пройтись по ним циклом
3. Скопировать внутренние массивы Arrays.copyOf

Сравнение и вывод
нужно использовать deepEquals и deepToString

Все обьекты в метод передаются по значению, но значением для массивов является ссылка
поэтому присвоение параматеру (массиву) новой ссылки не меняет исходный массив
Идиома безопасного копирования:
Возвращает метод тоже ссылку
поэтому в геттерах для инкапсуляции лучше возвращать копию массива, а то вернется исходник который можно изменить

varargs - синт сахар
Тип... имя
- только один такой параметр в методе
- используется только как аргумент метода и должен быть последним
- фактически является массивом
- желательно не создавать массив, а перечислить нужные аргументы

В java массивы коварианты (тип наследуется) - что опасно, т.к. проверка на совместимость происходит уже на этапе выполнения (не компиляуции)
Поэтому нельзя создавать массив параметризованного типа!
Object[] arr = new String[3];
arr[0] = 123; // ArrayStoreException
массив помнит свой исходный тип а инвариантный дженерик - стирает

Сборщик мусора не удаляет живой массив в обьекте, даже если на обьект уже нет ссылок
поэтому нужно занулять ссылки на более не нужный массив ненужного обьекта
loitering - удержание ссылок на неиспользуемый обьект

Массив можно заполнять лямбдой

Фактически массивы нужно использовать когда нужна:
- Производительность
- Работа с наборами примитивов
- Работа с низкоуровневым API
В остальных случаях предпочтительнее коллекции
